# 回答质量调优指南

## 可配置阈值一览

| 环境变量 | 默认值 | 含义 |
|---------|--------|------|
| `RAG_MIN_SCORE_STRICT` | 0.02 | 检索最高分低于此值 → 直接拒答，不调用 LLM |
| `RAG_MIN_SCORE_WARN` | 0.05 | 分数在此值和 STRICT 之间 → 生成回答但开头标注"置信度较低" |
| `INTENT_CONFIDENCE_THRESHOLD` | 0.5 | LLM 意图分类置信度低于此值 → 反问用户澄清 |

配置方式（`.env` 文件或环境变量）：
```bash
RAG_MIN_SCORE_STRICT=0.03
RAG_MIN_SCORE_WARN=0.08
INTENT_CONFIDENCE_THRESHOLD=0.4
```

## RAG_MIN_SCORE_STRICT / RAG_MIN_SCORE_WARN

这两个阈值控制"检索结果不可靠时是否调用 LLM 生成回答"。

### 行为矩阵

| 最高分 | 行为 |
|--------|------|
| < STRICT | ❌ 拒答：返回"未检索到可靠依据"，不调用 LLM（省 token 也防幻觉） |
| [STRICT, WARN) | ⚠️ 低置信生成：正常调 LLM，但 SYSTEM_PROMPT 注入"请标注置信度较低"指令 |
| >= WARN | ✅ 正常生成：完整回答 + 来源引用 |

### 调优建议

**RRF 分数说明：** 本项目使用 Reciprocal Rank Fusion（k=60），融合后的分数范围约为 0.0 ~ 0.1。
分数不是概率，而是排名信号。多个检索器都排在前面 → 分数更高。

| 场景 | 建议 STRICT | 建议 WARN |
|------|------------|-----------|
| 知识库文档质量高、覆盖面广 | 0.01（宽松，少拒答） | 0.03 |
| 知识库正在建设中，缺文档较多 | 0.03（严格，多提醒） | 0.06 |
| 电厂正式生产环境 | 0.02 | 0.05 |

**如何调参：**
1. 收集一批真实问题（50+）及其期望结果
2. 用不同阈值跑一遍，统计拒答率、低置信率
3. 目标：拒答率 < 15%，低置信率 < 30%
4. 如果某个阈值导致过多误拒 → 调低；导致过多低质量回答 → 调高

## INTENT_CONFIDENCE_THRESHOLD

当 LLM 分类的意图置信度低于此值时，系统不直接路由，而是反问用户。

### 行为

| 置信度 | 行为 |
|--------|------|
| >= 0.5 | 正常路由（mysql 或 rag） |
| < 0.5（且非关键词快速路径）| 🔄 反问：列出两种可能方向，等用户选择 |

### 调优建议

| 场景 | 建议值 |
|------|--------|
| 用户通常能清晰表达意图 | 0.4（宽松，少见反问） |
| 用户通常提问模糊（现场工人） | 0.6（严格，多反问确认） |
| 初期上线 | 0.5 |

## 关键词快速路径

以下查询**不经过**置信度判断，直接路由到 mysql：
- 包含关键词：`图纸` `下载` `调取` `查图` `找图` `dwg` `图档`
- 无论 LLM confidence 是多少

## 问题重写（指代消解）

当检测到以下信号时触发：
- 多轮对话中（`session_messages` 长度 > 1）
- 当前问题 ≤ 8 字符 或 包含代词（`这个` `那个` `它` `该设备` 等）

触发后额外消耗一次小额 LLM 调用（用于重写），重写后的问题覆盖原始问题进入后续流程。

## 审计日志新增字段

| 字段 | 说明 |
|------|------|
| `confidence` | 意图分类置信度（0-1） |
| `needs_clarification` | 是否触发反问 |
| `parse_status` | JSON 解析状态（success / retry_success / fallback） |
| `rewritten_query` | 重写后的问题（仅当重写触发时） |
| `max_retrieval_score` | 检索最高 RRF 分数 |

可通过 `scripts/query_audit_log.py` 查询以上字段：
```bash
python scripts/query_audit_log.py --from 2026-07-01 | jq '.confidence'
```
